TrueID :: Sycope Integration

Reference/example integration. CSV Lookup enrichment is the primary validated path. Custom Index injection depends on Sycope appliance support.

Enable Sycope Sync Push identity data to Sycope for NetFlow enrichment

Connection

Sycope Host URL of the Sycope appliance (e.g. https://sycope.example.com)
Username Sycope API user (Lookup rights required; Index inject only for Pattern B)
Password Not set
Test Connection Verify credentials and network connectivity

Pattern A — Lookup Enrichment

Active IP-to-identity mappings pushed to a Sycope CSV Lookup. Use in NQL: src stream="netflow" | set clientUser=lookup("TrueID_Enrichment", "user", {"ip": clientIp}, default="")

Lookup Name CSV Lookup name in Sycope (must exist in Sycope first)
Sync Interval How often to push data (seconds)

Pattern B — Event History

Optional and appliance-dependent. Auth and mapping change events are injected into a Sycope Custom Index only when the appliance/API supports it; otherwise Pattern B is auto-disabled at runtime and Pattern A continues in lookup-only mode.

Enable Event Index Inject auth events to custom Sycope index
Index Name Custom Index name (created via install.py)
